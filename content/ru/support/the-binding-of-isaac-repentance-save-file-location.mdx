Для эффективного управления этими путями игроки должны понимать роль файла `options.ini`. Изменение этого значения на `0` заставит игру игнорировать Steam Cloud и считывать данные только из локальной папки «My Games». Это важный шаг для игроков, предпочитающих ручное управление сохранениями или сталкивающихся с постоянными ошибками синхронизации.

<Steps>
  <Step num="1" title="Найдите свой Steam UserID">
    Перейдите в `C:\Program Files (x86)\Steam\userdata`. Если там только одна папка, это и есть ваш ID.   </Step>
  <Step num="2" title="Перейдите в папку Remote">
    Войдите в папку с вашим ID, затем найдите папку с именем `250900`. Внутри откройте папку `remote`. Это сердце вашего **расположения файлов сохранения the binding of isaac repentance**.

  <Step num="3" title="Скопируйте файлы .dat">

<Section icon={Swords} title="Устранение неполадок и FAQ">

Даже при правильном **расположении файлов сохранения the binding of isaac repentance** могут возникать ошибки типа «Corrupt Save» (Файл поврежден) или конфликты синхронизации. Большинство этих проблем связано с тем, как игра переключается между папкой `remote` и резервной папкой в `My Games`.

<Callout type="warning" title="Предупреждение о потере данных">
Никогда не удаляйте папку `250900`, если у вас нет подтвержденной резервной копии в другом месте. Удаление этой папки при активном Steam Cloud может привести к тому, что Steam синхронизирует «удаление», фактически стирая ваш прогресс и с облачных серверов тоже.
</Callout>

| Проблема | Потенциальное решение |
| :--- | :--- |
| **Сохранение не найдено** | Проверьте параметр `SteamCloud=1` в `options.ini` |
| **Сброс прогресса** | Переименуйте `rep_persistentgamedata`, чтобы номер соответствовал слоту |
| **Конфликт синхронизации** | Выберите «Загрузить в Steam Cloud», если локальный файл новее |
| **Вылет из-за мода** | Удалите папку `mods` в локальной директории |
